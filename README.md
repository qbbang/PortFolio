# 자기소개

### 기본정보

##### iOS로 전향하기 

전자공학과에서 디지털 회로 분야를 전공하여 제조업체에서 Mobile Camera Module 설계를 담당하다가

2014년에 평생 할 수 있는 일에 대해 생각하던 중, 아이디어를 곧바로 구현할 수 있는 SW에 흥미를 느꼈고,

SW 길을 걷고자, Automotive 컴파일러 기술지원 1년을 하고, 

Automotive 시장, LG전자에서 Qt Framework를 많이 다룬다는 것을 인지하고, 독학하여 주유기 업체에 취업하였고

Qt Framework를 이용하여 셀프 주유기 GUI 프로그램 개발 3년을 거쳤습니다.

　　QT를 사용하여 3년간 개발하면서,  개정으로 인해 처음 개발한 IC 전환 프로젝트 외 나머지 16개의 프로젝트는

기본 베이스에 정유사 업체 변경에 따른 UI 수정, VAN 업체 변경에 따른 리더기의 RS-232 통신 parsing 정보 수정,

POS 업체 변경에 따른 RS-422 통신 parsing 정보 수정과 같이,

살을 덧붙이거나 빼거나 하는 반복 작업 후 인증 심사 진행을 하다 보니,

SW를 시작하면서 처음 생각한 방향과 많이 다른 모습에, 좀 더 흥미롭고 주도적인 개발 

조금 더 사용자 경험을 사용자에게 제공하고,

이를 공식적으로 권장하고 강조하는 iOS에 흥미를 느끼게 되었습니다.


##### iOS 교육을 통한 실무 능력 향상

　iOS의 실무 능력을 키우기 위해서 iOS 교육을 받고 기본적인 문법 + 앱 개발의 기본소양을 배웠으며,

해커톤을 통해, "부득이한 정차 상황 시, 벌금을 최소화하자"라는 아이디어를 가지고,

강남구의 고정식 이동카메라의 위치 공동데이터를 가지고 반경 100m 안에 정차하게 되면,

5분 타이머를 작동하는 앱을 구현 했고,

백엔드 엔지니어와 협업을 통해, 배달의 민족 반찬 앱의 MVP 검색/장바구니/주문을 선정하고 이를 구현하면서

앱 서비스의 구현의 간접적으로 체험하였습니다.

##### iOS 인턴을 통한 실무 능력 향상

　　또한, 케어랩스의 굿닥 인턴과정을 통해, 실질적으로 공공데이터를 활용하여 구현한 앱서비스의 경험 했고,

그 곳에서, 네이버지도를 사용하는 클래스의 Objective-C 코드를 Swift로 변환하고, 더 이상 지원하지 않는

네이버지도 SDK 2.0를 3.0으로 변환하여 적용하는 실제 업무를 처리 하였습니다.

추가적으로 해당 앱서비스의 앱재사용율을 높히기 위한 기획을 인턴 과제로 받아, 병원/약국 찾기 외 사용자들이

앱을 주도적으로 사용할 수 있도록, 건강 관련 부가서비스를 기획하였고,

"간헐적 단식"이라는 아이템을 기획하고 디자인, 구현까지 진행하였습니다.

구현에 있어 API는 직접 설계하였고, 기획 바탕으로 프로토타입을 Adodb XD로 만들어 실제 구현에 참고하여

앱 내 부가서비스를 구현하였습니다.

주니어와 시니어 개발자의 차이는 경험이라고 생각합니다. 저의  iOS 개발자로서 나아가는데

좋은 양분이  믿어 의심치 않습니다.

### 성격 및 장단점

대체로 활발하고, 긍정적인 마인드를 가지고 있습니다.

좋아하는 배우로는 주성치와 짐캐리를 좋아하는데, 전혀 상반된 이미지 개그 또는 로맨스, 바보 또는 사이코패스, 

정극이든 사극이든 어떤 상황과 분위기에도 잘 어울려지는 그런 모습을 좋아합니다.

그래서 저의 마인드는 사람들과 소통을 쉽게 하기 위해서 장난기 있는 성격을 잘 살리려고 노력합니다.

과거 업무 중 결제 시스템 개발 시에 여러 VAN 업체, POS 업체들과 테스트를 하고 

개발을 진행 해야 하는 부분이 많았는데, 다른 업체 분들과 친밀감을 형성하다 보니, 급하게 처리하는 해야하는 일

심사 과정에서 사소한 문제 같은 경우 어느 정도 유도리 있게 처리과 되었습니다.

반대로 이러한 모습이 업무의 신뢰성을 낮추는 면도 있기 때문에,

raywenderlich 웹사이트를 통해, 지속해서 iOS에 대해 학습하고, 클린 코드를 베이스로 코드 한줄 한줄 

선배 개발자들의 장인 정신과 나의 개발 철학을 가지고 업무의 신뢰성을 쌓기 위해서 논리적인 사고력과 전문성을 

보여줄 수 있는 사람이 되고자 노력합니다.
