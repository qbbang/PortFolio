# 자기소개

### 기본정보

#### SW로 전향하기로 마음 먹다.

　전자공학과에서 디지털 회로 분야를 전공하여 제조업체에서 Mobile Camera Module 설계를 담당하다가 
 
평생 할 수 있는 일에 대해 생각하던 중, 아이디어를 곧바로 구현할 수 있는 SW에 흥미를 느끼게 되었습니다. 
 
Automotive 컴파일러 기술지원을 1년간 진행하면서, Automotive 시장에서 Qt Framework를 많이 다룬다는 것을 인지하게 되었습니다.

스스로 독학 후, 주유기 업체에 취업하게 되었고 Qt Framework를 이용하여 셀프 주유기 GUI 프로그램 개발 3년간 진행하였습니다.
 


---

#### 키오스트 SW를 개발하고 경험하다.

　셀프주유기 프로그램은 싱글톤으로 구현하였고, 화면을 관리하는 클래스를 이용하여, 각 뷰 클래스가 가지고 있는 static 변수에 주소를 가지고

이전 화면의 삭제하고 새로운 화면을 띄우는 방식으로 메모리 사용을 최소화하고, 최대한 단순하게 설계하고 작성하였습니다.

UI의 경우 기존 디자인에 포토샵을 이용해서 직접 수정하고 처리 하였습니다.

iOS 개발에서 다른 점은 분명 존재하지만, QT를 처음 공부하고 개발을 진행하였을 때, 항상 Qt Documentation 보고 필요한 메소드를

찾아보는 방식을 취했는데 iOS 역시 공식문서가 잘 정리되어 있어 학습하는데 조금 수월한 패턴으로 진행하였습니다.

또한, 필드에서 직접 의견을 수렴해서 커스텀하게 UI를 수정하면서, UX의 중요성을 직접적으로 체험했습니다.

UI/UX는 기획/디자이너의 역활도 크겠지만, 이전 경험을 통해 UI/UX에 대한 고민과 중요함을 얻은 것은

iOS 개발자로써 커다란 경험이라고 생각합니다.


---

#### 플랫폼을 전향하다.

　Qt Framework를 사용하여 3년간 기본 베이스에 정유사 업체 변경에 따른 UI 수정, VAN 업체 변경에 따른 리더기의 RS-232 통신, 
 
POS 업체 변경에 따른 RS-422 통신 parsing 정보 수정 등 살을 덧붙이거나 빼거나 하는 반복 작업 후 인증 심사를 진행하는 업무를 담당했습니다.
 
SW를 시작하면서 처음 생각한 방향과 달라 사용자 혹은 개인과 더 밀접하게 다가갈 수 있는 모바일에 대해 흥미가 생겼습니다. 
 
그중에서 사용자 경험을 중요하시며, 이를 공식적으로 권장하고 강조하는 iOS에 초점을 두게 되었습니다.


---


#### iOS 교육을 통한 실무 능력 향상시키다.

　iOS의 실무 능력을 키우기 위해서 iOS 교육을 받으며 기본적인 문법 + 앱 개발의 기본소양을 익혔습니다.
 
 해커톤을 통해 "부득이한 정차 상황 시 벌금을 최소화하자"라는 아이디어를 가지고 강남구의 고정식 카메라의 위치 공동데이터를 
 
 가지고 반경 100m 안에 정차하게 되면 5분 타이머를 작동되는 앱 구현의 결과물을 얻을 수 있었습니다. 
 
 또한 백엔드 엔지니어와 협업을 통해, 배달의 민족 반찬 앱의 MVP 검색/장바구니/주문을 선정하고 이를 구현하면서 앱 서비스의 구현에 대한 개념과 스킬을 얻었습니다.


---

#### iOS 인턴을 통한 실무 능력 향상시키다.

　케어랩스의 굿닥 인턴과정을 통해 실질적으로 공공데이터를 활용하여 구현한 앱서비스의 경험해볼 수 있었습니다.
 
　네이버지도를 사용하는 클래스의 Objective-C 코드를 Swift로 변환했고, 
 
 더 이상 지원하지 않는 네이버지도 SDK 2.0를 3.0으로 변환하여 실제 적용까지 진행했습니다. 

또한 해당 앱서비스의 개선을 위한 과제로 병원/약국 찾기 외 사용자들이 앱을 주도적으로 접속할 수 있도록 건강 관련 부가서비스를 기획하였습니다.

"간헐적 단식"이라는 아이템을 가지고 기획하고 API 설계, Adodb XD를 통해 앱 내 서비스가 구현되는 프로토타입을 제작하였습니다.



주니어와 시니어 개발자의 차이는 경험이라고 생각합니다.  iOS 개발자로서 나아가는데 좋은 양분이 믿어 의심치 않습니다. 


　
　
　

### 성격 및 장단점

활발하고, 긍정적인 마인드를 가지고 있습니다.

좋아하는 배우로는 주성치와 짐캐리를 좋아합니다. 개그 또는 로맨스, 바보 또는 사이코패스, 

정극이든 사극이든 상반된 다양한 상황과 분위기에도 잘 어울려지는 그런 모습이 저와 닮아 있다고 생각하기 때문입니다. 



그래서 본인 또한 사람들과 소통을 쉽게 하기 위해서 장난기 있는 성격을 잘 살리려고 노력합니다.

과거 업무 가운데 개발 시에 다양한  VAN 업체, POS 업체들과 테스트를 하고 개발을 진행 해야 하는 부분이 많았는데

다른 업체 분들과 친밀감을 형성하는 가운데 급하게 처 리하는 해야하는 일, 

심사 과정에서 사소한 문제 같은 경우 융통성 있게 처리할 수 있는 능력을 기를 수 있었습니다. 

반대로 이러한 모습이 업무의 신뢰성을 낮추는 면이 있을 수 있다고 생각합니다. 
 
선배 개발자들의 장인 정신과 본인의 개발 철학을 가지고 업무의 신뢰성을 쌓기 위해서 

raywenderlich 웹사이트를 통해 지속해서 iOS에 대해 학습하고, 

클린 코드를 베이스로 코드 한줄 한줄 리뷰 하는 등 논리적인 사고력과 전문성을 보여줄 수 있는 사람이 되고자 적극 노력하고 있습니다.
